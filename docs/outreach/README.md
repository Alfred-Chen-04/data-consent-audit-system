# Outreach 邮件 / DM 草稿

本文件夹是 SSRP 2026 项目的**用户 / 利益相关人调研**外联材料。所有文件都是**草稿**,不要直接发 — 每次发之前重读一遍,根据当时上下文微调。

## 目的

按"真实用户先用 → 学术 + 商业自然长出"的项目定位,在动手改架构 / 定 rubric 之前,先问**真实非学术用户**:你们用得上这种东西吗?

## 三条外联路线

| 路线 | 方式 | 为什么 | 预期回复率 |
|---|---|---|---|
| ① Dr. Singh 暖介绍 | 邮件问教授要人脉 | 教授一个引荐 >>> 10 封冷邮件,可信度最高 | >90% Singh 回,被引荐到的人 ~70% 回 |
| ② LinkedIn 冷 DM | 直接 connect + 一句话 | 绕过 `info@` 黑洞,找到具名人员,回复率比邮件高 2-3x | 20-40%(分人) |
| ③ 校友(LinkedIn Alumni filter) | 找本校校友现在在隐私 / regtech 公司的 | 校友纽带自带信任;但需要学校真的有这类校友 | 40-60% 校友场景下 |

## 发送前统一检查清单

对每一封都要过一遍:
- [ ] 对方名字 / 头衔 / 机构拼写正确
- [ ] 邮件主题不超过 70 字符,能一眼看懂你要什么
- [ ] 身份自证 3 句内完成(Dartmouth / SSRP / 项目一句话)
- [ ] **只问一件事**,不问"能不能聊 30 分钟"
- [ ] 给对方一个**低承诺 easy-out**("哪怕一两句话的想法也非常有用")
- [ ] Plan/proposal 链接就绪(Google Doc 或 PDF),不要附 .md 文件
- [ ] 如果要 CC Dr. Singh,事先想清楚 CC 的目的
